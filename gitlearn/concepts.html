Git Concepts

Understand how Git tracks history, manages branches, and enables collaboration.

How Git Works

Git tracks changes as a series of snapshots. Each commit points to a snapshot of your project and records metadata like author, date, and message.

Commits

A commit is a recorded snapshot of your project at a point in time. Commits form a history that you can traverse, revert, and share.

Commit structure

  • Message: Describes what changed.
  • Snapshot: The state of tracked files.
  • Parent: Reference to the previous commit.

Branches

Branches are pointers to commits. They let you work on new features without affecting the main codebase.

Branching in practice

Create a branch for each feature or fix, then merge it back when ready. This keeps history clean and makes code review easier.

Merging

Merging combines changes from one branch into another. Git automatically merges when possible but may require manual resolution if there are conflicts.

Merge conflicts

Conflicts happen when the same lines are changed in multiple branches. Resolve them by editing the file and then staging the result.

Remotes

Remote repositories are versions of your project stored on another server (e.g., GitHub). You can fetch, push, and pull changes between local and remote copies.

Common remote commands

  • git remote -v — List remote connections.
  • git fetch — Download new data from the remote.
  • git push — Upload local commits to a remote.
