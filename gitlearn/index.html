Git Basics

Git is a distributed version control system that helps developers track changes to files and collaborate with others.

What is Git?

Git stores snapshots of your project over time. It keeps a history of changes so you can revisit or revert to earlier versions when needed.

Why version control matters

  • Track who changed what and when.
  • Undo mistakes without losing work.
  • Collaborate safely with teammates.

How Git works

Git uses three main areas: the working directory, the staging area, and the repository. You edit files, stage them, then commit snapshots.

Troubleshooting

Git can sometimes behave in unexpected ways, but most issues can be resolved with a few helpful commands and best practices.

Common problems

  • Merge conflicts: Happens when Git can't automatically combine changes. Resolve conflicts by editing the files, then stage and commit.
  • Detached HEAD: Occurs when you checkout a specific commit instead of a branch. Switch back to a branch to continue working.
  • Accidental commits: Use git reset to undo commits locally before pushing.

Quick troubleshooting commands

  • git status — See what has changed.
  • git log --oneline — View commit history.
  • git diff — Compare changes.
  • git stash — Save work without committing.

Git Cheat Sheet

A quick reference for the most common Git commands.

  • git init — Start a new repository.
  • git clone <url> — Copy a remote repository locally.
  • git add <file> — Stage changes.
  • git commit -m "message" — Save staged changes.
  • git push — Send commits to remote.
  • git pull — Fetch and merge remote changes.
